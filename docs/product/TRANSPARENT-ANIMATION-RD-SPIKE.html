Vision

Celstate should create beautiful, robust transparent-background animations that can be dropped into any app, slide deck, editor, website, stream overlay, or social video without rotoscoping, chroma-key cleanup, or manual compositing.

We should not try to own broad video generation. We should own the production layer that turns generated motion into trustworthy transparent motion assets, regardless of whether the source is chroma RGB, native RGBA, a video model, or a third-party matting API. 

generated motion source
  + rough matte or native alpha
  + temporal and color evidence
  -> refined foreground RGB
  -> stable alpha
  -> artifact / confidence report
  -> WebM, ProRes MOV, APNG, and still exports

The first wedge is a short character or editorial overlay with soft edges, detached particles, motion blur, and coherent transparent padding for every frame.

Evidence

The flagship clip is no longer the main unknown. v7 is promoted on test3-trim1-sampled-green-s20; visual QA passed and exports verified as WebM VP9 alpha, ProRes 4444, and APNG.

The important progress is that quality is now measurable. The synthetic ground-truth eval compares compiler output to known RGBA truth, pins mean and worst-frame values, and catches regressions with pnpm alpha-eval run and pnpm alpha-eval compare.

Real-Prior Leg (BRIA, June 12 2026)

Same three truth scenarios, with the prior produced by BRIA RMBG (uvx rembg) over the decoded lossy frames instead of derived from truth. v6 ran against the identical captured prior frames via --prior dir, so the compiler comparison is apples-to-apples on the same imperfect prior. BRIA is evaluation evidence only; it is not a production fallback.

gt-sparks softBinarizationRate ↓ 0.53 0.69 0.75
gt-smoke residualSpill ↓ 0.021 0.098 0.124
gt-smoke softBinarizationRate ↓ 0.00 0.79 0.82
gt-tassels edgeAlphaMae ↓ 0.115 0.070 0.079
gt-tassels residualSpill ↓ 0.002 0.006 0.016
  • v7 beat v6 on every target metric under the real prior. The v7 mechanisms are not artifacts of the simulated prior.
  • Detached recall holds. 0.56 under BRIA vs 0.62 simulated — still far above v6 on either prior.
  • Genuine soft alpha is prior-bound. BRIA is a segmenter: it binarizes smoke at the prior level (softBinarizationRate 0.79 vs 0.00 simulated) and the compiler recovers only part of the loss. Soft smoke/glow content needs a matting-class prior — that is what the MatAnyone2 leg decides.
  • Thin strands got better, not worse. The crisp BRIA prior beat the blurred simulated prior on tassel edge fidelity (0.070 vs 0.115) — the simulated prior's blur was the limiter there.
  • Metrology note: priorAlphaMae alone understates prior damage on soft content because background pixels dominate the mean; read it together with softBinarizationRate.
Honest limitation The MatAnyone2 leg has not run yet (run it on Windows WSL2 with CUDA; ingest via --prior dir --prior-alpha-dir). The BRIA baseline (baselines/synthetic-eval-bria.json) is machine-pinned evaluation evidence, not a portable CI gate and not a runtime path; the CLI refuses cross-prior comparisons.

What Worked

  • Existing generated videos are good enough for R&D. Do not spend effort producing new provider clips unless a local alpha experiment proves the current sources cannot answer the question.
  • MatAnyone2 temporal prior + chroma detached-element fusion is the right shape for the flagship wedge. The temporal prior stabilizes the subject; chroma evidence recovers separated leaves and particles the prior drops.
  • Projection decontamination beat channel-clamp despill. v6 repaired mixed fringe RGB along the background-to-reference axis instead of merely desaturating edges.
  • v7 moved the hard generalization metrics. Color-line alpha fusion, matting-equation foreground recovery, evidence-gated detached recovery, reference seed filtering, and background plate erosion all contributed measurable gains.
  • Exports are not the current blocker. The harness verifies WebM VP9 alpha, ProRes 4444, APNG, stills, preview composites, heatmaps, and reports.
  • Truth-referenced eval changed the process. Generalization claims now require metrics on deterministic RGBA scenarios, not just visual inspection of one clip.

What Did Not Work

Attempt Result Enduring Lesson
Global chroma-key similarity sweeps Rejected A key radius broad enough to catch compressed backgrounds also deletes green-adjacent foreground.
Temporal background plate without strong exclusion Rejected Moving foreground contaminates the plate and creates subject ghosts.
Simple chroma matting equation on fringe Rejected Ambiguous partial-alpha pixels can turn into cyan, pink, or doubled edge artifacts.
Channel-clamp / pull-to-inward despill Superseded It desaturates contaminated pixels but cannot reconstruct foreground color.
MatAnyone2 fgr/ as clean foreground RGB Rejected fgr/ is source composited over a fixed pale-green plate, not decontaminated foreground.
GP-01 / GP-02 generalization probes Smoke only They test plumbing, not quality: GP-01 misses detached recovery and GP-02 is mostly binary rather than genuine soft smoke.

Active Plan

  1. Run the MatAnyone2 prior leg. BRIA leg done v7 held on the real BRIA prior. Generate alpha outputs for the three scenario source.mp4 files on Windows WSL2/CUDA, ingest with --prior dir, and decide the soft-content stance: BRIA-class priors binarize smoke, so soft alpha needs MatAnyone2 or it does not ship.
  2. Start production-worker design for the proven paths. Subject + detached content under chroma sources is now evidence-backed end to end (flagship visual QA, simulated truth, real prior). The production design should require the chosen prior and fail closed on prior failure rather than downgrade to another model.
  3. Broaden the scenario matrix. Add different key colors, larger soft glows, denser particles, non-mascot overlays, text-safe lower thirds, and slide/app insertion checks.
  4. Keep native-RGBA providers as comparators. If a provider outputs excellent alpha directly, Celstate still needs benchmarking, QA, export hardening, and workflow UX.

Decision Criteria

This brief should always make the state obvious. Update this section whenever the evidence changes.

A. Making Progress

Use when we have a measured improvement path, the next experiment is clear, and the expected result would materially improve product confidence.

Current state: yes.

B. Stuck

Use when real-prior or broader-scenario evidence fails and we no longer have a concrete mechanism likely to move quality metrics or user-visible reliability.

Current state: no.

C. Vision Achieved

Use only when transparent animations consistently pass visual QA, truth metrics, real-prior evaluation, export verification, and drop-in tests across apps, slides, editors, and web surfaces.

Current state: no.

Operating Notes

  • Canonical harness: scripts/spikes/transparent-animation-spike.ts, with run artifacts under tmp/transparent-animation-spike/runs/<run-id>/.
  • Canonical eval: scripts/spikes/alpha-compiler/, with deterministic scenarios gt-sparks, gt-smoke, and gt-tassels.
  • Promoted artifact: tmp/transparent-animation-spike/review/test3-trim1-sampled-green-s20/.
  • Default compiler: v7. Do not resurrect older despill/keying paths except as baselines.
  • Production reliability rule: no legacy fallback ladder. If MatAnyone2 becomes the production prior, the worker must make that path reliable, observable, and fail closed when it cannot produce a trustworthy prior.
  • Windows/WSL2 path: MatAnyone2 should be tested on the actual target machine. The upstream repo supports Python 3.10, editable install or uv, video input plus first-frame mask, and CUDA device inference.
  • BRIA caveat: BRIA/rembg exists only to stress-test the compiler against an imperfect real prior. Keep it out of production-worker design unless we make a deliberate future product decision with the same reliability bar.
  • Production boundary: this is still an R&D brief, not a product spec or production worker plan.
