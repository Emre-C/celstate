Product thesis

Celstate already creates transparent still assets. The animation opportunity is the same promise with motion: short transparent assets creators can drop into OBS, editors, social video, and websites without rotoscoping, chroma-key cleanup, or manual compositing.

Celstate should not own video generation broadly. Celstate should own the conversion of generated motion into trustworthy transparent motion assets — regardless of upstream model (Veo, Runway, Kling, native-RGBA, third-party matting APIs). 

generated RGB / chroma video
  + rough matte or native alpha
  + temporal signals
  → refined foreground RGB
  → stable alpha video
  → confidence / artifact report
  → WebM / ProRes MOV / APNG / still exports

First wedge: A humanoid mascot crosses a transparent canvas, turns back with personality, and wind-blown leaves or particles move coherently — exported as OBS/editor-ready alpha video.

R&D verdict

We are not blocked on video generation for the flagship wedge. Existing generated videos are good enough for transparency R&D. Do not generate new provider videos unless a local experiment proves current sources cannot answer the question.

The Alpha Compiler thesis is credible on the flagship clip: MatAnyone2 temporal prior + chroma detached-element fusion, projection decontamination (v6), and export plumbing all work. That clip is no longer the primary unknown.

The primary unknown is generalization: detached particles, genuine soft partial alpha, and fine repeated structures. Phase 1 measurement confirms v6 still fails these cases even with an ideal simulated prior — the mechanism gaps are real, scoped, and now regression-checkable.

v6 architecture

source.mp4 + MatAnyone2 pha/ + sharp chroma alpha
  → per-frame sure-background plate (outward fill from empty pixels)
  → subject inward reference fill from deep core
  → detached interior reference fill from chroma-opaque interiors
  → projection decontamination: out = src − t·(bg − ref)
  → subject fringe + core band use subject ref; detached edge band uses detached ref
  → QA metrics + spill heatmap + export

Key lessons baked into v6:

  • Projection decontamination replaces channel-clamp despill — reconstructs mixed fringe RGB instead of desaturating it.
  • Detached interiors preserve source color exactly; only edge bands are repaired.
  • MatAnyone2 fgr/ is not clean foreground — it is source composited over a fixed pale-green plate. Do not use as an RGB prior.

v7 architecture (promoted)

source.mp4 + MatAnyone2 pha/ + sharp chroma alpha
  → reference seeding from key-free observations (filtered by core proximity)
  → per-frame background plate filled to closure (outward from empty + eroded near fg evidence)
  → color-line alpha estimate per pixel: src = a·ref + (1−a)·bg
  → fuse baseline alpha with color-line estimate (weight capped, upward corrections gated)
  → matting-equation foreground recovery: fg = src + ((1−a)/a)·(src − bg), gain capped
  → evidence-gated detached path (color evidence, not hard distance)
  → QA metrics + spill heatmap + export

Key mechanisms added in v7:

  • Color-line alpha fusion — per-pixel two-color model estimates alpha where the prior is soft or missing, with confidence-weighted blending capped at COLOR_LINE_MAX_WEIGHT.
  • Matting-equation foreground recovery — inverts straight-alpha compositing against the per-frame background plate, with gain capping to prevent noise amplification at low alpha.
  • Evidence-gated detached path — re-adds detached elements based on color evidence rather than hard distance, preserving soft alpha where the prior deleted particles.
  • Reference seed filtering — key-free observations are only used as reference seeds when sufficiently distant from the prior core, preventing contaminated thick-structure edges from polluting the fill.
  • Background plate erosion — sure-background seeds are eroded near foreground evidence so faint halos cannot pollute the background plate used for matting recovery.

Synthetic ground-truth eval (Phase 1)

Canonical generalization evidence. Truth-referenced metrics compare compiler output against known RGBA ground truth — none are tautological.

Module layout

scripts/spikes/alpha-compiler/
  core.ts              pure v6 numerical core (unit-tested, no I/O)
  truth.ts             deterministic synthetic RGBA generators
  metrics.ts           truth-referenced per-frame metrics + worst-frame aggregation
  baseline.ts          baseline build / compare with explicit tolerances
  eval-cli.ts          pipeline CLI (pnpm alpha-eval)
  baselines/synthetic-eval.json   committed regression baseline
  alpha-compiler.test.ts          49 unit tests

Pipeline (per scenario)

  1. Generate RGBA truth frames (deterministic seeded PRNG)
  2. Composite over noisy chroma plate
  3. H.264 yuv420p round trip (CRF 23)
  4. FFmpeg colorkey chroma alpha
  5. Simulated matting prior (truth alpha minus detached elements, Gaussian-blurred)
  6. createV6RgbaFrame
  7. Compare output against stored truth

Honest limitation: simulated prior The matting prior is derived from truth (blurred, detached elements zeroed), not MatAnyone2/BRIA output. This isolates the compiler deterministically but does not measure real prior failure modes. A real-prior eval leg is required before production confidence.

Failure taxonomy

Active defect classes — map to metrics and v7 levers.

Failure Symptom Metric / lever

Foreground erosion Loose keying deletes subject interior falseTransparentRate · do not solve via similarity sweeps alone
Detached binarization Particles re-added as hard blobs with green rings detachedAlphaRecall, softBinarizationRate · leaf path, edge bands

Temporal alpha flicker Frame-to-frame alpha breathing unrelated to motion temporalAlphaInstability · stabilization (post-v7 spatial fixes)
Prior vs compiler confusion Good matte, bad RGB — or vice versa priorAlphaMae vs compiler metrics · separate eval legs

Solved for spike: export compatibility (WebM VP9 alpha, ProRes 4444, APNG). Remaining work is quality, not plumbing.

Closed experiments

These directions were tried and are not open work:

  • Global similarity / despill-mix sweeps — erodes foreground or leaves halos; rejected.
  • Chroma matting equation on fringe (v3) — creates cyan/pink ghosts; rejected.
  • Channel-clamp / pull-to-inward despill (v5) — desaturates instead of reconstructing; superseded by projection.
  • MatAnyone2 fgr/ as RGB prior — source-over-green composite, not decontaminated RGB; rejected.
  • Per-frame BRIA-only mattes — useful fallback, not the architecture; MatAnyone2 + chroma fusion wins on flagship.

Escalation rule

  • Celstate owns prior-fusion, projection decontamination, color-line fusion, matting-equation recovery, QA metrics, and export hardening.
  • If a native-RGBA provider fully solves quality with no compiler layer, shift R&D to benchmarking, workflow UX, and export QA.
  • Per-frame BRIA remains the fallback for probes and environments without MatAnyone2.
