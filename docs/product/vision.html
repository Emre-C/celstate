Product Vision & Status

Celstate

AI transparent-background image generator. Describe anything — a logo, a character, a product shot — and get it back already transparent. No background removal, no halos, no edge degradation. Just your vision, clean and ready to use.

Last updated: June 25, 2026

North Star

Celstate is the fastest path from imagination to a production-ready transparent image. The core technical differentiation is difference matting — a mathematically exact alpha recovery technique that generates the same subject on white and black backgrounds, then solves for perfect transparency. No ML-based background removal, no edge artifacts.

The product extends from static images to transparent vector animations (Lottie JSON), positioning Celstate as the transparent-asset layer for creatives, designers, and AI agents.

Product Thesis

Transparent images. Nothing else. Professionals in focused productivity mode arrive with intent, not to browse. They need transparent-background images and want to describe what they need and get it fast. The app should respect their time and reward their focus.

The business model is equally simple: start free (3 credits + weekly drip), buy credit packs when you need more ($5 or $10 one-time). No subscriptions, no expiration, no complexity.

Target User & Core Promise

  • Who: Creatives, designers, marketers, and AI agents who need transparent PNGs — logos, characters, product shots, icons, stickers.
  • Promise: Describe it. Generate it. Already transparent.
  • Quality bar: 32-bit RGBA PNGs with a proper alpha channel. Edges that hold up on any background. No halos, no color bleed, no fringing.
  • Speed bar: Type a prompt, get your image. No queues, no multi-step workflows.

Current App Shape

The app is a SvelteKit frontend with a Convex backend, deployed on Vercel. Authentication uses Clerk with Convex custom JWT validation. Image generation runs through Vertex AI (gemini-3.1-flash-image-preview) in a staged durable pipeline. Credits and payments are handled via Stripe. A remote MCP server exposes tools for agentic clients.

  • Frontend: SvelteKit 5, TailwindCSS 4, light-mode only, warm parchment palette
  • Backend: Convex (real-time sync, mutations, actions, cron jobs, HTTP actions)
  • AI: Vertex AI via @google/genai (image generation), Gemini structured output (Lottie)
  • Auth: Clerk (Google live, Apple when enabled) + Convex JWT template
  • Payments: Stripe Checkout (one-time credit pack purchases)
  • Deploy: Vercel (frontend) + Convex (backend), with automated deploy verification
  • Observability: PostHog (analytics), Sentry (exceptions), Discord/Slack webhooks (ops alerts)

Major Product Gaps

  • Assisted Mode not implemented: Spec exists for a designer LLM layer that refines prompts before generation, but no code has been written. Not beta-blocking.
  • No live generation demo on landing: Prospective users cannot try the product without signing up. A interactive demo could improve conversion.
  • No A/B testing infrastructure: Landing page copy and CTA placement are unvalidated. No experiment framework exists.
  • Apple sign-in not enabled: Google is live, but Apple requires dashboard configuration. Not blocking but limits sign-in options.
  • Lottie render QA missing: Users can receive structurally valid but visually broken animations. Acceptable for V1 internal, blocks paid surface.
