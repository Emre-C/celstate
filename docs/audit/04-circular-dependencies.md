# Audit 04: Untangle Circular Dependencies

**Date:** 2026-06-24  
**Scope:** `src/`, `packages/`, `scripts/`  
**Tool:** `madge` (v8.0.0)

---

## Summary

Madge found **20 circular dependencies** in `src/`. All 20 are false positives caused by Convex's auto-generated `api.d.ts` file. **Zero real circular dependencies exist** in the codebase. This was verified by re-running madge with `--exclude "convex/_generated/.*"`, which reported no cycles. `packages/` and `scripts/` were also checked separately and are clean.

---

## Findings

### All 20 cycles: `convex/_generated/api.d.ts` false positives

Every cycle follows the same pattern:

```
convex/_generated/api.d.ts → convex/<module>.ts → [intermediate hops] → back to api.d.ts
```

`api.d.ts` is auto-generated by `pnpm exec convex codegen` and contains type references to every Convex function. When a Convex module imports `internal` or `api` from `_generated/api.js`, madge sees a cycle: `api.d.ts → module.ts → api.d.ts`.

**Verification:** Re-running with `--exclude "convex/_generated/.*"` confirms zero cycles:

```
$ npx madge --circular --extensions ts --exclude "convex/_generated/.*" src/ --ts-config tsconfig.json
Processed 128 files (1.6s)
✔ No circular dependency found!
```

Cycles 1-3 have longer chains (`creditPackPurchase.ts → lifecycle.ts → users.ts → posthog.ts`) but are still false positives — the cycle only closes through `api.d.ts`. Without the generated file, `creditPackPurchase.ts → lifecycle.ts → users.ts → posthog.ts` is a **linear dependency chain**, not a cycle. `posthog.ts` does not import back to any upstream module.

**Verdict:** All 20 are false positives. This is a known madge limitation with generated code. No action needed.

---

## Coupling Observation (Not a Cycle)

While not a circular dependency, the `creditPackPurchase` module's dependency chain is worth noting:

```
creditPackPurchase.ts → lifecycle.ts → users.ts → posthog.ts
```

`lifecycle.ts` imports `applyCreditsToUser` from `users.ts`, and `users.ts` imports `posthog` for analytics. This creates tight coupling between the credit pack purchase module and the user/analytics modules. It is not a cycle and poses no runtime risk, but extracting `applyCreditsToUser` into a standalone `src/convex/lib/credits.ts` module would improve separation of concerns and shorten the dependency chain.

---

## Critical Assessment

The codebase has **no circular dependencies**. All 20 madge reports are artifacts of Convex's generated `api.d.ts` file. The dependency graph is acyclic when generated files are excluded. The longest dependency chain (`creditPackPurchase → lifecycle → users → posthog`) is linear and poses no initialization-order risk.
